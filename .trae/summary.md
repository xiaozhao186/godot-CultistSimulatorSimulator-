# 更新总结：属性值整型化与配置修正

根据您的反馈，我们进行了以下修复和优化：

## 1. 属性数值类型更改为整型 (Int)

为了符合您的需求，我们将所有属性数值的比较和存储逻辑更改为了整数类型，不再使用小数。

- **EventCondition**: `value` 字段现在是整数类型。在 Inspector 中输入时将不再有小数点。
- **EventPanel**: 统计卡牌属性时，会强制转换为整数进行累加。
- **CardData**: 虽然字典本身支持多种类型，但建议您在配置卡牌属性时，数值统一使用整数（例如 `1` 而不是 `1.0`）。

## 2. 问题排查：条件无法识别的原因

您遇到的 "无法识别" 问题，根据您的截图和日志分析，**是因为属性名称不一致导致的**。

- **Debug 日志显示**：卡牌拥有的属性是 `货币` (Currency)。
  `Total Stats: { &"货币": 1.0 }`
- **截图显示**：您在条件 (Conditions) 中填写的 Attribute 是 `金钱` (Money)。
  `Attribute: 金钱`

**解决方法**：请确保卡牌数据 (`CardData`) 中的属性名与事件分支条件 (`EventBranchData`) 中的属性名完全一致。请将条件中的 Attribute 改为 `货币`，或者将卡牌属性改为 `金钱`。

## 3. 下一步建议

由于修改了 `EventCondition` 的脚本结构（`value` 变为 `int`），Inspector 中已有的数据可能会显示异常或丢失。请您：
1. 打开相关 `EventData`。
2. 检查 `Branches` -> `Conditions`。
3. 重新输入数值（确认没有小数点）。
4. **修正属性名**（统一为 "货币" 或 "金钱"）。
