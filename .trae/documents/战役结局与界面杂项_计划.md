# 战役结局与界面杂项：实施计划

## 目标概述

实现四类功能改动：

1. **战役结局系统**：在事件奖励中触发“结局编号”，游戏暂停并弹出不可关闭的结局界面（标题/正文/图片 + 返回主菜单）。
2. **Verb 隐形判定机**：Verb 可配置为场上不可见且不可交互，但其内部事件仍可自动运行（用于状态判定/终局触发器等）。
3. **事件面板体验优化**：

   * 开局/运行中的 auto\_start 事件不强制弹出面板（默认后台运行）。

   * 面板隐藏状态下的 force inhale：吸入卡牌应与面板状态同步隐藏，不应“漂浮在空中”。

   * 进入结算（COLLECTING）时面板应自动弹出显示，避免奖励在“空中”出现。
4. **Scenario 初始牌数量**：ScenarioData 初始牌支持“某卡牌 x N 张”，减少重复配置。

* 扩展 [EventRewardData.gd](file:///d:/Program%20Files%20\(x86\)/Godot_v4.5.1/box/ppd-1/scripts/EventRewardData.gd)：

  * `@export var ending_index: int = -1`（-1 表示不触发）

  * （可选）`@export var pause_on_ending: bool = true`（默认暂停）

### 1.4 触发时机与执行路径

* 在 [EventPanel.gd](file:///d:/Program%20Files%20\(x86\)/Godot_v4.5.1/box/ppd-1/scripts/EventPanel.gd) 的奖励处理阶段（目前在 `_process_finished_cards` 的 rewards loop）：

  * 若 `reward.ending_index >= 0`：调用 `GameManager.trigger_ending(ending_index)`（只传索引，不在事件里直接加载资源）。

  * 该结局事件通常会设置 `auto_start + auto_collect`，确保无需玩家操作即可走到触发点。

### 1.5 GameManager 增加“当前战役”上下文与触发 API

* 在 [GameManager.gd](file:///d:/Program%20Files%20\(x86\)/Godot_v4.5.1/box/ppd-1/scripts/GameManager.gd)：

  * 记录 `current_scenario: ScenarioData`（在 `start_scenario` 时赋值；在 `restore_game_state` 根据存档的 `scenario_id` 反查并赋值）。

  * 增加方法：`trigger_ending(index: int)`：

    * `AppState.set_paused(true)`（保证 Timer 等逻辑跟随暂停策略）

    * 从 `current_scenario.endings[index]` 取资源，交给 UI 展示

    * 显示结局面板（不可关闭），只提供“返回主菜单”按钮

### 1.6 结局 UI（不可关闭）

* 新建场景 `EndingPanel.tscn` + 脚本 `EndingPanel.gd`：

  * UI：标题 Label、正文 RichTextLabel、图片 TextureRect、按钮 “返回主菜单”

  * 行为：

    * 显示时 `mouse_filter = STOP`，阻止底层交互

    * 不提供关闭按钮

    * 点击返回：`AppState.set_paused(false)`（或保持暂停但切场景前解除）→ `change_scene_to_file("res://scenes/MainMenu.tscn")`

  * 挂载方式：参考 `SettingsPanel` / `DebugConsole`，放到 `Tabletop.tscn` 的 CanvasLayer 下或由 `GameManager.trigger_ending` 动态实例化。

## 功能 3：事件面板体验优化

### 3.1 auto\_start 默认后台运行（不强制弹出面板）

* 目标：开局或运行中由 auto\_start 触发的事件，不要强制弹出面板打断玩家。

* 调整点：

  * `Verb._check_auto_start` 不直接调用 `_on_clicked` 产生可见面板，而是创建面板后设置 `visible=false`（等价于“最小化启动”）。

  * 允许玩家之后点击 Verb 时再显示该面板（当前逻辑：`active_panel` 存在时点击会 `visible=true`）。

### 3.2 面板隐藏时的 force inhale：吸入卡牌同步隐藏

* 目标：面板被最小化/隐藏时，任何被 force inhale 吸入 slot 的卡牌都应隐藏（而不是停留在 slot 世界坐标“空中”）。

* 调整点：

  * 在 `EventPanel._attempt_magnet_inhale` 完成吸入后：

    * 若 `not visible`：将 `chosen.visible=false` 且 `input_pickable=false`（与 `_hide_force_inhaled_cards` 保持一致）。

  * 保留现有 `visibility_changed` 机制：面板再次显示时 `_show_force_inhaled_cards` 恢复可见性（但保持锁定）。

### 3.3 进入结算（COLLECTING）自动弹出面板

* 目标：当事件完成进入结算态，若面板此前隐藏，应该自动弹出以展示奖励堆与收集按钮。

* 调整点：

  * 在 `_enter_collection_mode` 开始处执行：

    * `visible = true`

    * `move_to_front()`

## 验收/测试清单（手动）

1. 结局触发：

   * 在某事件的 `EventRewardData.ending_index` 设为 1，事件设置 `auto_start + auto_collect`。

   * 触发后应暂停，出现结局面板（不可关闭），显示对应 scenario 的 endings\[1]。

   * 点击“返回主菜单”成功返回，且桌面交互不残留。
2. 隐形 Verb：

   * 配置 `hidden_runtime=true` 的 VerbData，确认桌面不显示且不可点。

   * 其 default\_event 若 `auto_start`，应在后台运行并能触发后续逻辑（例如吸入终局卡触发 ending）。
3. auto\_start 面板行为：

   * 开局 auto\_start 不弹窗；点击 Verb 可打开并看到正在运行的面板。
4. force inhale 隐藏同步：

   * 面板隐藏状态下仍吸入卡牌，但卡牌应消失（不漂浮）；再次打开面板卡牌正确显示在 slot。
5. 结算自动弹窗：

   * 面板隐藏时事件结束进入 COLLECTING，应自动显示面板并展示奖励堆。
6. 初始牌数量：

   * `initial_card_entries` 中设置某卡 `count=5`，开局生成 5 张；旧 `initial_cards` 仍可用。

## 风险与兼容性

* 数据资源扩字段（ScenarioData/VerbData/EventRewardData）会导致旧资源在编辑器里显示新字段为空：通过默认值保证兼容。

* 结局 UI 需要在暂停状态仍可交互：结局面板节点应设置 `process_mode = ALWAYS`（参照 SettingsPanel/DebugConsole/Verb/EventPanel 已用策略）。
