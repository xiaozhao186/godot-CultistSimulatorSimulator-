# Phase 2: Core Features & Creation Workflow Implementation Plan

This plan addresses the implementation of Phase 2 features (branching support) and the new requirement for creating events directly within the editor with automated folder organization.

## 1. Branching & Conditions Support
Enhance `EventNode` and `GraphView` to handle complex branching logic.

- **Update `scenes/event_node.tscn` & `.gd`**:
  - Dynamically add output slots based on the `branches` array in `EventData`.
  - Add a dedicated slot for `instant_branches` (Recipe Preview).
  - Visualize slots with different colors to distinguish branch types.
- **Update `scenes/graph_view.gd`**:
  - Handle connections to specific branch indices.
  - When connecting a "Branch" slot to another node:
    - If the branch exists, update its `target_event`.
    - If it's a new connection on a dynamic port, create a new `EventBranchData` entry.
  - Implement disconnection logic for branches.

## 2. In-Editor Event Creation (New Requirement)
Implement context menu and logic to create new `EventData` resources directly in the graph.

- **Update `scenes/graph_view.gd`**:
  - Add a `PopupMenu` for context actions (Right-click).
  - **Menu Option: "Create New Event"**:
    - Show a confirmation/naming dialog (or default to "New Event X").
    - **Logic**:
      1. Determine save path: `res://data/Events/<GraphName>/<EventName>.tres`.
         - Create the folder `res://data/Events/<GraphName>/` if it doesn't exist.
      2. Instantiate new `EventData`.
      3. Save it to disk using `ResourceSaver`.
      4. Add it to the graph immediately at mouse position.
- **UI Update**:
  - Add the `PopupMenu` node to `GraphView.tscn`.
  - Connect `gui_input` on `GraphEdit` to trigger the menu.

## 3. Inspector Integration (Condition Editing)
Allow editing of branch conditions when selecting a connection or node.

- **Strategy**:
  - Since Godot's GraphEdit connections aren't "selectable objects" easily inspected, we will primarily support editing conditions via the **Event Node's Inspector**.
  - When an `EventNode` is selected, the Inspector should show the `EventData` resource.
  - Ensure `EventBranchData` sub-resources are editable within that Inspector view.
  - (Optional/Advanced) Add a small "Edit" button next to output slots in `EventNode` to popup a specific condition editor dialog. For this phase, we'll rely on the standard Inspector which is already robust for Resources.

## 4. Verification & Testing
- **Test Branching**:
  - Create Event A and Event B.
  - Add a branch to Event A via Inspector (or dynamic slot if implemented).
  - Connect Event A (Branch 0) -> Event B.
  - Verify `EventA.branches[0].target_event` is Event B.
- **Test Creation**:
  - Open graph `MyQuest.tres`.
  - Right-click -> "Create New Event".
  - Verify file created at `res://data/Events/MyQuest/NewEvent.tres`.
  - Verify node appears and is valid.
