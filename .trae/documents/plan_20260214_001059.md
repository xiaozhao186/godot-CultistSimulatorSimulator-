# Phase 3: In-Graph Event Editing Implementation Plan

This plan focuses on integrating `EventData` editing capabilities directly into the Graph Editor, reducing the need to switch to the standard Inspector dock.

## 1. Inspector Integration (EditorInterface)
Instead of building a custom UI for every property, we can leverage Godot's built-in `EditorInspector`.

-   **Goal**: When a node is selected in `GraphEdit`, the standard Inspector dock should automatically show that node's `EventData` resource.
-   **Implementation**:
    -   Update `scenes/graph_view.gd`: Connect to the `node_selected` signal of `GraphEdit`.
    -   In the signal handler, use `EditorInterface.edit_resource(selected_node.event_data)` to force the Inspector to display the resource.
    -   *Note*: This is the most "Godot-native" way and requires minimal code while providing full editing power (including arrays, sub-resources, etc.).

## 2. Quick-Edit Side Panel (Optional/Advanced)
If the user prefers a dedicated side panel within the graph view (to avoid looking away to the far right):

-   **Goal**: Create a custom "Properties" panel inside the `GraphView` scene.
-   **Implementation**:
    -   Update `scenes/graph_view.tscn`: Add a `HSplitContainer` to divide the view into `GraphEdit` (Left) and `VBoxContainer` (Right/Properties).
    -   Inside the Property panel, use `EditorInspector` (via `EditorInterface` magic) or manually build a form for common fields (`id`, `title`, `description`, `duration`).
    -   *Decision*: For this phase, we will prioritize **Method 1 (Standard Inspector Integration)** as it's more robust. If requested, we can add a simple "Quick Edit" overlay for `ID/Title/Duration` on the node itself or a small side panel.

## 3. Dynamic Node Updates
Ensure that changes made in the Inspector are reflected immediately in the Graph Node (e.g., changing `id` updates the node title).

-   **Implementation**:
    -   In `scenes/event_node.gd`, connect to the `changed` signal of the `event_data` resource.
    -   On `changed`: Update `title`, `Label` text (duration), and rebuild slots if `branches` or `instant_branches` arrays changed size.
    -   *Constraint*: Godot's `Resource.changed` signal is sometimes spammy or doesn't trigger on deep sub-resource changes. We might need to manually trigger updates or use `setget` in `EventData` (modifying original script) or check on `_process` / input events.
    -   *Refinement*: To avoid modifying `EventData.gd` runtime script, we will rely on the `changed` signal. If that's insufficient, we'll add a "Refresh" button or check on mouse enter.

## 4. Execution Steps
1.  **Inspector Link**: Implement `_on_node_selected` in `graph_view.gd` to call `EditorInterface.edit_resource()`.
2.  **Live Updates**: Implement `_on_data_changed` in `event_node.gd` and connect it in `setup()`.
3.  **Branch Refresh**: Ensure modifying the `branches` array in Inspector triggers `_rebuild_slots()` in `EventNode`.

## 5. Verification
-   Select a node -> Inspector shows `EventData`.
-   Change `duration` in Inspector -> Node label updates.
-   Add a new item to `branches` in Inspector -> Node gains a new Orange output port.
