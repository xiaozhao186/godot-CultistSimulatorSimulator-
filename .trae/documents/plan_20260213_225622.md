# Phase 1: Event Chain Editor Prototype Implementation Plan

This plan outlines the steps to create the initial prototype of the Event Chain Editor plugin for Godot, focusing on visualizing and editing linear event chains using `default_next_event`.

## 1. Directory Structure & Plugin Initialization
Create the foundational file structure for the plugin in `addons/`.

- **Create Directory**: `addons/event_chain_editor/`
  - Subdirectories: `resources/`, `scenes/`, `assets/`
- **Create `plugin.cfg`**: Define plugin metadata (name, description, author, version, script).
- **Create `event_chain_editor.gd`**: The main `EditorPlugin` script.
  - Implement `_enter_tree()` to register the `EventGraphData` custom type and load the main UI dock.
  - Implement `_exit_tree()` to clean up.
  - Implement `_handles()`, `_edit()`, and `_make_visible()` to handle opening `EventGraphData` resources.

## 2. Data Layer Implementation
Define the resource that stores the graph's layout data.

- **Create `resources/event_graph_data.gd`**:
  - Extends `Resource`.
  - **Properties**:
    - `nodes`: Dictionary (Maps `event_resource_path` (String) -> `position` (Vector2)).
    - `scroll_offset`: Vector2 (Stores graph view position).
    - `zoom`: float (Stores graph zoom level).

## 3. UI Component: Event Node
Create the visual representation of a single event in the graph.

- **Create `scenes/event_node.tscn`**:
  - Root node: `GraphNode`.
  - Child nodes: `Label` (Title/ID), `Label` (Duration/Type).
- **Create `scenes/event_node.gd`**:
  - Extends `GraphNode`.
  - **Properties**: `event_data` (Reference to the `EventData` resource).
  - **Functionality**:
    - `setup(data: EventData)`: Initialize UI from resource.
    - **Slots**:
      - Left Slot 0: Input (Enable logic).
      - Right Slot 0: Output (Connects to `default_next_event`).

## 4. UI Component: Graph View
Create the main editor interface.

- **Create `scenes/graph_view.tscn`**:
  - Root node: `Control`.
  - Child node: `GraphEdit`.
- **Create `scenes/graph_view.gd`**:
  - Extends `Control`.
  - **Functionality**:
    - `load_graph(graph_data: EventGraphData)`: Clear existing nodes, instantiate `EventNode` for each entry in `graph_data`, restore positions and connections.
    - `save_graph()`: Iterate through nodes, update `EventGraphData` with positions, and update actual `EventData` resources with connection info.
    - **Connection Handling**:
      - `_on_connection_request`: Connect nodes visually and update `EventData.default_next_event`.
      - `_on_disconnection_request`: Disconnect visually and clear `EventData.default_next_event`.
    - **Node Management**:
      - Context menu to "Add New Event" or "Add Existing Event".
      - Support dragging `.tres` files from FileSystem (if feasible in prototype, otherwise use context menu).

## 5. Integration & Verification
Connect the pieces and verify functionality.

- **Integration**:
  - Ensure the plugin loads without errors.
  - Verify `EventGraphData` appears in the "Create New Resource" menu.
  - Verify double-clicking an `EventGraphData` file opens the Graph Editor in the bottom panel.
- **Verification Steps**:
  1. Create a new `EventGraphData` resource.
  2. Open it.
  3. Create two nodes (representing two `EventData` resources).
  4. Connect Node A to Node B.
  5. Save the graph.
  6. Inspect Node A's `EventData` resource to confirm `default_next_event` now points to Node B.
  7. Close and reopen the graph to confirm layout restoration.
