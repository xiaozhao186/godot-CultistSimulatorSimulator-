# 判定机无吸入检测与调试系统改进：实施计划

## 目标
完成 3 类改动：
1. **判定机不移动卡牌**：在不改变“Verb + Event 充当判定机”的基础上，实现对桌面卡牌（进程卡/终局卡等）的状态检查与分支判断，但不吸入/不移动场上卡牌。
2. **DebugConsole 改进**：
   - 增加“加载预先设置的行为（VerbData）”功能（区别于“把 Event 绑定到一个临时 Verb”）。
   - 适配事件资源按文件夹分剧本存放后的数据层级，便于筛选/检索。
3. **Debug 输出开关/集中管理**：在设置面板增加“是否输出 debug 信息”的开关，并逐步把高频 debug 输出收敛到一个统一脚本，避免后续影响流畅度。

## 现状要点（已确认）
- 事件分支判断当前基于“slot 内卡牌聚合统计”，因此判定机常用做法是 force inhale 把卡牌吸到 slot 再判定（导致卡牌短暂消失）。
- `DebugConsole` 当前只扫描 `EventData/CardData` 并提供“绑定事件 → 生成一个新的 Verb”的调试操作；并没有扫描/加载 `VerbData` 的入口。  
  关键实现点：`DebugConsole._scan_data()`、`DebugConsole._on_bind_pressed()`
- 目前 debug 输出主要依赖 `print()`/`push_warning()` 散落在脚本中；设置面板只有“是否显示 DebugConsole”，没有“是否输出 debug 日志”的全局开关。

## 功能 1：判定机不移动卡牌（全局状态检查）

### 方案选择（采用）
新增一组“基于桌面卡牌”的条件类型（Tabletop 条件），用于事件分支判断：
- **TABLE_HAS_CARD_ID**：桌面上存在指定 `CardData.id`
- **TABLE_HAS_TAG**：桌面上存在指定 tag
- **TABLE_COUNT_TAG / TABLE_COUNT_CARD_ID**：桌面上满足条件的数量满足比较（>=、== 等）
- （可选）**TABLE_SUM_ATTRIBUTE**：对桌面上全部卡牌某数值属性求和后比较（用于“进度值/士气值”等全局指标）

这些条件只做“读取与判断”，不会移动卡牌，也不依赖 slot。

### 数据结构改动
- 扩展 [EventCondition.gd](file:///d:/Program%20Files%20(x86)/Godot_v4.5.1/box/ppd-1/scripts/EventCondition.gd)：
  - `Type` 增加上述枚举项
  - 增加字段：`card_id: String`（用于按 card id 判断；对 tag/attribute 仍复用现有字段）
  - 更新 `_validate_property`：根据 Type 显示/隐藏 `tag/attribute/card_id/value/op` 相关字段

### 运行时逻辑改动
- 扩展 [EventBranchData._check_condition](file:///d:/Program%20Files%20(x86)/Godot_v4.5.1/box/ppd-1/scripts/EventBranchData.gd)：
  - 当 cond 为 TABLE_* 类型时，改为从 Tabletop 遍历 Card 节点，做存在/计数/求和判断（复用 `_compare`）
  - 与现有 HAS_TAG/SUM/COUNT 保持并存：slot 条件继续基于 slots；TABLE 条件基于桌面

### 判定机使用方式（剧本侧）
- 判定机事件不再需要 force inhale slot 来“搬运卡牌”，而是配置分支条件为 TABLE_HAS_CARD_ID / TABLE_HAS_TAG 等。
- Verb 仍可 `hidden_runtime + auto_start` 后台运行该事件；事件进入 auto_collect 后自动触发奖励（生成/删除 Verb、触发结局等）。

### 验收
- 桌面放置“进程卡/终局卡”时，判定机事件无需吸入卡牌即可正确导向分支/触发奖励。
- 取消/移走进程卡时，判定机事件能在下一轮判断里恢复正常分支。

## 功能 2：DebugConsole 改进（加载预设行为 + 数据层级适配）

### 2.1 增加 VerbData 扫描与加载
- 修改 [DebugConsole.gd](file:///d:/Program%20Files%20(x86)/Godot_v4.5.1/box/ppd-1/scripts/DebugConsole.gd)：
  - `_scan_data()` 同时收集 `VerbData`
  - UI 增加一个 `VerbData` 的下拉选择 + “生成预设行为”按钮
  - 点击后直接实例化 `Verb.tscn` 并 `verb.setup(verb_data)`（等价于 Scenario 初始行为生成方式）

### 2.2 事件资源按文件夹分剧本的筛选/检索
改进 DebugConsole 的事件选择方式（两步）：
1. 扫描时记录每个 EventData 的 `res_path`（或由 ResourceLoader 获取路径），以便按路径前缀分组。
2. UI 增加一个“剧本/目录过滤”选项：
   - 例如以 `res://data/feceva/Events/<剧本名>/` 作为一级分类
   - 二级再选择事件
3. （可选增强）增加搜索框：按事件 id/title/路径子串过滤列表

### 验收
- DebugConsole 可直接“生成预设 VerbData”（不是仅绑定事件）。
- 面对大量事件资源，能通过剧本/目录快速定位到目标事件。

## 功能 3：Debug 输出开关与集中管理

### 3.1 增加全局开关（设置面板）
- 扩展 [AppState.gd](file:///d:/Program%20Files%20(x86)/Godot_v4.5.1/box/ppd-1/scripts/AppState.gd)：
  - 新增 `debug_logs_enabled`（默认 false 或沿用当前习惯：debug console 关时日志也关）
  - 提供 getter/setter + signal
- 扩展 [SettingsPanel.gd](file:///d:/Program%20Files%20(x86)/Godot_v4.5.1/box/ppd-1/scripts/SettingsPanel.gd)：
  - 新增一个 CheckBox：“输出 Debug 日志”
  - 切换时调用 `AppState.set_debug_logs_enabled(...)`

### 3.2 新增统一日志脚本并逐步迁移高频输出
- 新增 `scripts/Log.gd`（autoload 单例）：
  - `debug(msg)` / `warn(msg)` / `error(msg)`
  - `debug()` 仅当 `AppState.debug_logs_enabled` 为 true 时才输出（用 `print` 或 `print_rich`）
  - warn/error 可选择不受开关影响（或也受影响）
- 第一阶段只迁移高频 `print("[Debug] ...")` 输出点：
  - `EventPanel.gd`、`Verb.gd`、`GameManager.gd` 的大量 `[Debug]` 语句替换为 `Log.debug(...)`
  - 保留 `push_error/push_warning` 用于真正错误

### 验收
- 关闭“输出 Debug 日志”后，运行时不再刷大量 `[Debug]` 输出。
- 开启后能恢复输出，便于开发期定位问题。

## 风险与兼容性
- EventCondition 新增 Type 会影响编辑器资源展示：通过 `_validate_property` 确保字段显隐正确，避免误填。
- Tabletop 条件需要访问桌面节点：统一使用 `get_tree().root.get_node_or_null("Tabletop")` 避免路径漂移；若未来改场景名，需要同步更新入口。
- Debug 日志迁移不必一次做完：先迁移高频点即可显著减压。
