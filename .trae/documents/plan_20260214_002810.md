# Phase 4: Full In-Node Editing Implementation Plan

This plan aims to implement a fully editable UI directly within the `GraphNode`, removing the dependency on the Inspector for common tasks.

## 1. UI Redesign of EventNode

We will transform `EventNode` from a simple label container into a complex control panel.

* **Header Area**:

  * `LineEdit`: Editable **ID** (updates `resource_name` and `id`).（id is directly bound to the name of the created .tres file）

  * `LineEdit`: Editable **Title**.\
    （还有可编辑文本text，别忘加上）

* **Properties Area** (VBoxContainer):

  * `SpinBox`: **Duration** (float).

  * `CheckBox`: **Auto Start**.

  * `CheckBox`: **Auto Collect**.

* **Branch Management Area**:

  * **Instant Branches Section**:

    * Header Label + "Add" Button.

    * Dynamic Rows: Each row contains a Slot (Port) + "Edit Condition" Button + "Delete" Button.

  * **Result Branches Section**:

    * Header Label + "Add" Button.

    * Dynamic Rows: Each row contains a Slot (Port) + "Edit Condition" Button + "Delete" Button.

* **Rewards Area**:

  * Simple "Edit Rewards" Button (pops up a dedicated dialog or relies on Inspector for complex reward arrays, or we add a basic list if feasible).

## 2. Logic Implementation

Update `scenes/event_node.gd` to handle these UI interactions.

* **Data Binding**:

  * `_on_id_changed(new_text)`: Update `event_data.id`, rename resource file (optional/advanced, maybe just rename property).

  * `_on_duration_changed(value)`: Update `event_data.duration`.

  * `_on_auto_start_toggled(pressed)`: Update `event_data.auto_start`.

* **Branch Operations**:

  * `_on_add_branch_pressed(type)`:

    * Create new `EventBranchData`.

    * Append to `branches` or `instant_branches`.

    * Trigger `_rebuild_slots()` (which will now render the new row with controls).

  * `_on_delete_branch_pressed(index, type)`:

    * Remove from array.

    * Trigger `_rebuild_slots()`.

* **Condition Editing**:

  * `_on_edit_condition_pressed(branch_data)`:

    * Open a `AcceptDialog` containing a property editor for the `conditions` array of that branch.

    * *Simpler approach for v1*: Just use `EditorInterface.edit_resource(branch_data)` to focus Inspector on just that branch.

## 3. Saving Mechanism

Ensure changes persist.

* Connect all UI signals (`text_changed`, `value_changed`, `pressed`) to a method that:

  1. Updates the `EventData` resource in memory.
  2. Calls `ResourceSaver.save()` immediately (or marks dirty).
  3. Emits `changed` signal so the graph view knows (if needed).

## 4. Execution Steps

1. **Scene Update**: Modify `event_node.tscn` to add the new Controls (LineEdits, SpinBoxes, Buttons).
2. **Script Logic**: Rewrite `event_node.gd` `setup()` and `_rebuild_slots()` to populate these controls and connect their signals.
3. **Branch Management**: Implement the Add/Remove logic directly in the node script.
