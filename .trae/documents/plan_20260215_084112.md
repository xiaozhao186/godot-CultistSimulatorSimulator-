# Phase 6: Final Stabilization & Slot Editing Implementation

This plan addresses the persistent "read-only array" issue and implements the requested slot editing features.

## 1. Fixing "Array is in read-only state" Permanently

The error occurs because new Resources created in-memory often share empty arrays as references or are marked as read-only until modified explicitly.
**Solution**:

* In `EventNode.setup()`, we will explicitly check `branches`, `instant_branches`, `rewards`, and `slots`.

* If any array is read-only OR shared (same reference as a default), we will `duplicate(true)` it immediately.

* This ensures that as soon as the node UI loads, the data is safe to modify.
