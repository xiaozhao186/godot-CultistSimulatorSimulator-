- 主场景：[Tabletop.tscn](file:///d:/Program%20Files%20(x86)/Godot_v4.5.1/box/ppd-1/scenes/Tabletop.tscn)

## 已实现功能点对照
- 桌面背景：`res://pic/background.jpg` 已作为桌面底图加入主场景
- 相机边界：相机位置根据背景尺寸 + 当前 zoom + 分辨率做 clamp
- 卡牌边界：拖拽与放下都会对卡牌位置做 clamp，保证卡牌不越出桌面内缩边界
- 重叠弹开：卡牌放下时仅核心区（约 88%）重叠触发弹开
- 动态检测开关：卡槽内卡牌 `overlap_enabled=false`，释放回桌面自动恢复
- 拖动缩放：拖起 1.12 倍，放下回 1.0，tween 0.22s
- 卡槽判定：从“鼠标点命中”改为“卡牌-槽位矩形交叠比例”
  - 放入阈值：≥ 60%
  - 静止确认：200ms 内位移 ≤ 2px
  - 预览：拖拽时最佳候选槽位比例 ≥ 45% 高亮

## 关键参数（可调）
- 桌面内缩边距：`TableBounds.card_margin_ratio`（默认 0.08）
  - 位置：主场景节点 `TableBounds` 的导出属性
- 拖动缩放：`Token.DRAG_SCALE`（默认 1.12）
- 重叠核心区比例：`GameManager._get_card_core_rect()` 内 `0.88`
- 弹开力度：`resolve_card_overlaps()` 内 `min_push = core_half.x * 0.18`
- 卡槽阈值：`EventPanel.gd` 内 `best_ratio < 0.6` 与 `preview >= 0.45`

## 快速验收步骤（手工）
1. 相机边界
   - 按住 WASD / 中键拖拽平移 + 滚轮缩放
   - 观察相机是否始终不露出桌面背景外的“空白”
2. 卡牌边界
   - 将卡牌拖向桌面四角，松开
   - 观察卡牌是否始终完整留在桌面范围内
3. 重叠弹开
   - 让多张卡牌核心区重叠后松开
   - 观察是否会自然分离，且不会抖动/飞出桌面
4. 卡槽放置准确性
   - 将卡牌拖到槽位边缘附近（约半遮挡）
   - 观察槽位高亮预览是否出现；松开后 200ms 内不动才真正“吸附放入”
